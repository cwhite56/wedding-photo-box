# Wedding Photo Dropbox

## Overview
This project is my take on an alternative to event photo dropbox products. Rather than pay for one of these services, I decided to create my own. The project uses a REST API architecture to handle HTTP requests from the frontend. A QR code is generated for guests to scan which routes directly to the dropbox page. Once the photos are sent, they are saved to the server's hard drive and a path to the photo is saved to the PostgreSQL database for retrieval. 
## Purpose
The purpose of this project was to learn how to use Spring Boot and implement RESTful services. I learned the basics of Spring Boot, how to implement a REST API, how to handle files as JSON objects from requests, and the industry standard controller -> service -> repository architecture to handle my domain objects. This project was focused on backend development. The frontend index.html was generated by AI. I plan to go back and learn those langauages in the future.
## Usage
To use this project, the mobile device must be connected to the same wifi as the host computer. `docker compose up` is ran to start the database. `mvn spring-boot:run` starts the server. The QR code png file is created to be distributed. From there, users can add photos from their camera roll and send them directly to the host computer.
### Status
This project is functional when used over the same wifi network. I plan to add AWS connectivity so that it can run without needing a host computer on the same wifi. I will be adding a feature that allows an admin to bulk download the files using the file paths saved to the database.
